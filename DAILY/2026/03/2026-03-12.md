Quanto aos accepts, dá-nos jeito para:
- Data exfiltration precisa de accepts com BytesSent
- Um accept de src interno para dst interno num porto incomum → Lateral Movement (?)

Campos no project:
- `NATSourceIP` e `VirtualDomain` -


**Event Disrupt 1 Hour -** Não precisamos nada em específico, refere apenas à ausência de logs.
**Possible Port Scan / Host Scan -** Precisamos de `deny`/`drop`
**High Number of Events -** Count de eventos por source. Precisamos de todos os actions para ser significativo, ou pelo menos accepts + denies
**Abnormal Outbound Traffic -** Precisamos de `accept` com `BytesSent` e `DestinationCountry`. Este UC justifica ingerir accepts com campos de bytes.
**Outbound Traffic to Dangerous Country -** Precisa de `accept` com `DestinationCountry`. Denies para países perigosos são menos interessantes, ou que importa é o que passou.
**External Traffic via Insecure Protocols -** Precisa de `accept` com `Application` ou `DestinationPort` - HTTP, Telnet, FTP, etc que passaram
**External RDP/SSH Connections Allowed -** Precisa de `accept` com `DestinationPort 3389/22` e `DestinationCountry` externo
**SYSTEM Login / Configuration Outside Business Hours -** Estes são logs de audit, não traffic, e não têm sido um problema no seu volume.


```kql
// ============================================================
// Fortinet Traffic Transform
// DCR: dcr-layer8-fortinet-traffic
// Tabela destino: Fortinet_CL
// Scope: Apenas eventos que alimentam analytic rules ativas
// Bundle: SOC8 UseCases Bundle - Firewall (Traffic + Audit)
//
// Nota arquitetural: o ELK recebe todos os logs Fortinet
// sem filtro — serve como repositório completo para hunting,
// IR e retenção. O Sentinel só ingere o que tem analytic
// rule ativa associada, minimizando custos.
// ============================================================

source

// ------------------------------------------------------------
// PRÉ-EXTRAÇÃO DE CAMPOS USADOS NOS FILTROS
// Campos necessários nos `where` abaixo têm de existir
// antes de serem referenciados — limitação das DCR transforms.
// Extraídos aqui em extends separados antes dos filtros.
// ------------------------------------------------------------
| extend Action             = extract(@"act=([^\s]+)", 1, RawData)
| extend DestinationCountry = extract(@"dstcountry=([^\s]+)", 1, RawData)
| extend DestinationPort    = toint(extract(@"dpt=([^\s]+)", 1, RawData))

// ------------------------------------------------------------
// FILTRAR EVENTOS SEM ACTION
// Logs de sistema do FortiGate sem act= (ex: HA sync,
// config events) não têm Action e não alimentam nenhum
// use case de traffic — descartados aqui.
// ------------------------------------------------------------
| where isnotempty(Action)

// ------------------------------------------------------------
// FILTRAR APLICAÇÕES SEM VALOR DE SEGURANÇA
// Protocolos de infra e monitoring que geram volume alto
// mas não alimentam nenhuma analytic rule do Default Bundle.
// !has usa term index — mais eficiente que !contains,
// que faz substring scan caractere a caractere.
// ------------------------------------------------------------
| where RawData !has "app=DNS"
    and RawData !has "app=NTP"
    and RawData !has "app=DHCP"
    and RawData !has "app=Ping"
    and RawData !has "app=ICMP"
    and RawData !has "app=SNMP"
    and RawData !has "app=mDNS"
    and RawData !has "app=NBNS"
    and RawData !has "app=Google-DNS"
    and RawData !has "app=Cloudflare-DNS"
    and RawData !has "app=Microsoft-Azure"
    // Zabbix agent (10050) e server (10051) — monitoring interno
    // FortiGate pode formatar com TCP_ ou tcp/ dependendo da versão
    and RawData !has "app=TCP_10051"
    and RawData !has "app=tcp/10051"
    and RawData !has "app=TCP_10050"
    and RawData !has "app=tcp/10050"

// ------------------------------------------------------------
// FILTRAR SESSÕES DE FECHO SEM CLASSIFICAÇÃO DE APP
// close/server-rst/timeout com appcat=unscanned são sessões
// muito curtas ou protocolos não reconhecidos com terminação
// limpa — sem classificação de app, o valor de segurança
// é mínimo e o volume é alto.
// Condição AND deliberada: se app estiver classificada,
// o evento passa mesmo sendo close/rst (tem contexto útil).
// ------------------------------------------------------------
| where not(RawData has "act=close"      and RawData has "appcat=unscanned")
    and not(RawData has "act=server-rst" and RawData has "appcat=unscanned")
    and not(RawData has "act=timeout"    and RawData has "appcat=unscanned")

// ------------------------------------------------------------
// FILTRO PRINCIPAL POR USE CASE — Default Bundle
//
// deny / drop — sempre ingeridos:
//   → Possible Port Scan [FW_FONTE]
//   → Possible Host Scan [FW_FONTE]
//   → High Number of Events [FW_FONTE]
//   → Event Disrupt 1 Hour [FW_FONTE] (baseline de volume)
//
// accept — apenas nas condições abaixo:
//
//   DestinationCountry externo (não Reserved/private):
//   → Outbound Traffic to Dangerous Country [FW_FONTE]
//   → External RDP/SSH Connections Allowed [FW_FONTE]
//   → [REPORT] External Traffic via Insecure Protocols [FW_FONTE]
//
//   Portos inseguros ou de acesso remoto:
//   → [REPORT] External Traffic via Insecure Protocols [FW_FONTE]
//     porto 20/21 FTP, 23 Telnet, 69 TFTP, 80/8080 HTTP
//   → External RDP/SSH Connections Allowed [FW_FONTE]
//     porto 22 SSH, 3389 RDP
//
//   BytesSent > 10MB por sessão:
//   → Abnormal Outbound Traffic [FW_FONTE]
//     threshold ajustável conforme baseline do cliente
//
// Accepts HTTPS internos rotineiros (maior volume) não
// satisfazem nenhuma condição e são descartados.
// O ELK tem esses dados disponíveis para hunting e IR.
//
// Nota: `in` com inteiros não é suportado em DCR transforms.
// Substituído por comparações explícitas por porto.
// ------------------------------------------------------------
| where (Action == "deny" or Action == "drop")
    or (
        Action == "accept"
        and (
            // Tráfego para destinos externos — país classificado
            // "Reserved" = RFC1918 e ranges especiais no FortiGate
            // "private" = redes internas não geolocalizadas
            (
                isnotempty(DestinationCountry)
                and DestinationCountry != "Reserved"
                and DestinationCountry != "private"
            )
            // Portos inseguros — HTTP, FTP, Telnet, TFTP
            or DestinationPort == 20    // FTP data
            or DestinationPort == 21    // FTP control
            or DestinationPort == 23    // Telnet
            or DestinationPort == 69    // TFTP
            or DestinationPort == 80    // HTTP
            or DestinationPort == 8080  // HTTP alternativo
            // Portos de acesso remoto
            or DestinationPort == 22    // SSH
            or DestinationPort == 3389  // RDP
            // Transferências grandes — proxy de exfiltração
            // Segundo extract necessário porque BytesSent
            // ainda não existe neste ponto do pipeline
            or (
                RawData has "out="
                and toint(extract(@"out=([^\s]+)", 1, RawData)) > 10000000
            )
        )
    )

// ------------------------------------------------------------
// EXTRAÇÃO DE CAMPOS
// Todos os campos necessários para as analytic rules do bundle.
// DestinationCountry e DestinationPort já extraídos no topo —
// o extend aqui não os redeclara, apenas completa o schema.
// toint() usado em bytes/pacotes por constraint do schema
// da tabela (campos definidos como Int na Fortinet_CL).
// Para usar tolong() é necessário alterar o schema primeiro.
// NATSourceIP necessário para correlação com sistemas externos
// que só veem o IP público após NAT.
// VirtualDomain necessário se cliente tiver VDOMs configurados.
// ------------------------------------------------------------
| extend
    Message           = RawData,
    LogID             = extract(@"logid=([^\s]+)", 1, RawData),
    PolicyID          = toint(extract(@"policyid=([^\s]+)", 1, RawData)),
    PolicyName        = extract(@"policyname=([^\s]+)", 1, RawData),
    SourceIP          = extract(@"src=([^\s]+)", 1, RawData),
    SourcePort        = toint(extract(@"spt=([^\s]+)", 1, RawData)),
    DestinationIP     = extract(@"dst=([^\s]+)", 1, RawData),
    Protocol          = toint(extract(@"proto=([^\s]+)", 1, RawData)),
    Application       = extract(@"app=([^\s]+)", 1, RawData),
    Duration          = toint(extract(@"duration=([^\s]+)", 1, RawData)),
    BytesSent         = toint(extract(@"out=([^\s]+)", 1, RawData)),
    BytesReceived     = toint(extract(@"in=([^\s]+)", 1, RawData)),
    PacketsSent       = toint(extract(@"sentpkt=([^\s]+)", 1, RawData)),
    PacketsReceived   = toint(extract(@"rcvdpkt=([^\s]+)", 1, RawData)),
    InboundInterface  = extract(@"deviceInboundInterface=([^\s]+)", 1, RawData),
    OutboundInterface = extract(@"deviceOutboundInterface=([^\s]+)", 1, RawData),
    SourceCountry     = extract(@"srccountry=([^\s]+)", 1, RawData),
    NATSourceIP       = extract(@"sourceTranslatedAddress=([^\s]+)", 1, RawData),
    NATSourcePort     = toint(extract(@"sourceTranslatedPort=([^\s]+)", 1, RawData)),
    Severity          = extract(@"deviceSeverity=([^\s]+)", 1, RawData),
    DeviceName        = extract(@"dvchost=([^\s]+)", 1, RawData),
    VirtualDomain     = extract(@"vd=([^\s]+)", 1, RawData)

// ------------------------------------------------------------
// OUTPUT FINAL
// Schema explícito — evita colunas inesperadas da source
// se o schema mudar. RawData excluído porque Message = RawData.
// DestinationCountry e DestinationPort incluídos aqui apesar
// de extraídos no topo — necessário para aparecerem no output.
// ------------------------------------------------------------
| project
    TimeGenerated,
    Message,
    LogID, PolicyID, PolicyName,
    SourceIP, SourcePort,
    DestinationIP, DestinationPort,
    Action, Protocol, Application, Duration,
    BytesSent, BytesReceived,
    PacketsSent, PacketsReceived,
    InboundInterface, OutboundInterface,
    SourceCountry, DestinationCountry,
    NATSourceIP, NATSourcePort,
    Severity, DeviceName, VirtualDomain
```
